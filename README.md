## ✨🚀 Welcome to `wbt-summer-2025-2026`!

This repository serves as a dynamic workspace for the "Web Technologies" course, spanning the summer terms of 2025-2026. It's designed to host a collection of practical lab tasks, coding exercises, and progressive project development, culminating in a personal portfolio. This journey begins with fundamental client-side interactions, such as robust form validation using vanilla JavaScript and the DOM API, laying a strong foundation for advanced web development concepts.

---

## ✨ Key Features

*   ✅ **Client-Side Form Validation**: Implement robust input checks directly in the browser.
*   🔒 **Essential Data Integrity**: Validate critical user inputs like name, email, phone number, and age.
*   🚫 **Real-time User Feedback**: Provide immediate alerts for invalid or incomplete form submissions.
*   🎯 **DOM Manipulation**: Directly interact with HTML elements to read and validate form data.
*   🏗️ **Structured Learning Environment**: A dedicated space for organized lab tasks and project development.

## ⚡ Getting Started

Follow these steps to get a local copy of the project up and running on your machine.

### Prerequisites

You only need a modern web browser to view and interact with the project files.
*   Any modern web browser (e.g., Chrome, Firefox, Edge, Safari).

### Installation

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/yourusername/wbt-summer-2025-2026.git
    ```
2.  **Navigate into the project directory:**
    ```bash
    cd wbt-summer-2025-2026
    ```

### Running the Project

To view the current lab tasks:

1.  **Open the HTML files directly in your browser.**
    For instance, to view `Lab Task 5.html`:
    ```bash
    # On macOS/Linux
    open "Lab Task 5.html"

    # On Windows
    start "Lab Task 5.html"
    ```
    Alternatively, you can simply double-click the `Lab Task 5.html` file in your file explorer.

## 📝 Usage

The current `Lab Task 5.html` demonstrates a basic user registration form with JavaScript-based client-side validation.

*   **Fill out the form fields:** `Name`, `Email`, `Phone`, and `Age`.
*   **Attempt to submit:** Click the "Submit" button.
*   **Observe validation alerts:**
    *   An alert will appear if any field is empty.
    *   The email field requires an `@` and `.` character.
    *   The phone number must be exactly 11 digits long.
    *   The age must be 18 or older.
*   **Successful submission:** If all conditions are met, an "Form submitted successfully!" alert will appear.
